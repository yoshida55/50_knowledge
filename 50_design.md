## 2026-05-18 LinkWorks（就労継続支援A型サイト）

### ヒーロー画像の選定
- **決定**: スタイリッシュな3人写真 → 水彩画（スタッフ＋利用者のPC場面）に変更
- **理由**: ターゲット（就労支援を考えている人・家族）が自分を重ねやすい画像にする必要があった。おしゃれ優先だとターゲットが「自分の話ではない」と感じる
- **教訓**: ヒーロー画像は「かっこいいか」より「見た人が自分事と感じるか」で選ぶ

### スライド2枚構成
- **決定**: 1枚（水彩・PC業務）→ 2枚（PC業務＋軽作業）のスローフェード切り替え
- **理由**: LinkWorksの2本柱（軽作業・IT業務）を自然に伝えられる。同じ水彩スタイルなので切り替わっても印象がブレない
- **教訓**: スライドショーはメッセージがブレるのでNG。ただし同じトーン・同じ目的の画像なら有効

### フラッシュ演出
- **決定**: 切り替わり時に白フラッシュ（opacity 0.3、1.2s）を追加
- **理由**: 切り替えの瞬間を「演出」として見せることでサイトの完成度が上がる
- **数値メモ**: opacity 0.6は眩しすぎ → 0.3がちょうどいい

### 装飾テキストの扱い（"WORKS WITH YOU" / "About us"）
- **決定**: 青ベタ → `rgba(23, 27, 32, 0.08)` の薄いグレーに変更
- **理由**: 装飾テキストは「ここを読んでください」という意図がない場所。主張すると視線を奪い、本来見てほしいコンテンツの邪魔になる
- **教訓**: 装飾テキストは opacity 0.06〜0.10 程度が適切。色をつけると途端に主張しすぎる

### company.html ヒーローの2列レイアウト
- **決定**: テキスト（左60%）＋画像（右40%）のグリッド。テキスト先に出てから画像がワイプイン
- **理由**: 画像に目が行きすぎると理念テキストが読まれない。テキスト側を広くして「まず読ませる」設計にする
- **アニメーション順**: テキスト0.2s → 画像1.0s（左が先に出て右が後から開く）

### 就労支援サイトのターゲット設計
- **原則**: 文章・画像のターゲットは「利用を考えている人・家族」に絞る
- **企業へのアピール**: テキストで直接訴求するのではなく、サイト自体の完成度・設計の丁寧さで間接的に示す
- **参考**: Re-start（BtoB特化）は企業・事業所向けにビジネス映像が正解。LinkWorksとはモデルが違うので参考にするときは注意

### サービスセクションのカードアニメーション
- **決定**: transition を 1.6s → 2.6s に延長。中央カードの遅延を 0.18s → 0.8s に
- **理由**: 動きが速すぎると「見えた気がする」だけで終わる。ゆっくりにすることで動きを意識して楽しんでもらえる
- **教訓**: 演出として見せたいアニメーションは遅くする。遅延をずらすと順番が見えて気持ちいい

### company.html テキストカードの浮かせデザイン
- **決定**: テキストブロックを `background:rgba(255,255,255,0.88)` + `border-radius:2rem` + `box-shadow` でカード化し、`align-self:center` で縦方向にフロートさせる
- **理由**: テキストが画像列の高さいっぱいに引き伸びてしまうと間延びして見える。カードを独立させることで「テキストエリアとして読む」流れが生まれる
- **教訓**: グリッドの子要素は `align-self:center` を忘れると親の高さいっぱいに伸びる。カード化するなら必ずセットで確認する

### ヘッダーの被り対策（padding-top）
- **決定**: `.co_hero` に `padding-top:8rem` を追加してヘッダー分のスペースを確保
- **理由**: `position:fixed` ヘッダーはフロー外のため、セクション先頭のコンテンツが重なる。padding-top で押し下げるのがシンプルで確実
- **教訓**: fixed ヘッダーがあるサイトでは、各ページのファーストビューに padding-top をつけるのをデフォルトにしておく

### グラデーションの採用と却下
- **決定**: テキスト側にグラデーション背景を検討したが、採用せず
- **理由**: 「画像に目が行きすぎる」問題の解決策としてグラデーションを試みたが、最終的にグリッド比率（6:4）変更と白カード効果で十分に解決できた。グラデーションは余分な視覚的ノイズになる
- **教訓**: レイアウト問題をグラデーションで解決しようとするのは最終手段。まず構造・比率で解決できないか先に試す

### アニメーション出現順序の設計
- **決定**: テキスト（delay:0.2s）→ 画像のワイプイン（delay:1.0s）の順
- **理由**: 画像が先に出ると視線が右（画像）に奪われ、左のメッセージが読まれない。テキストを先に着地させてからビジュアルを開くことで「読む → 見る」の流れを作る
- **教訓**: 2列レイアウトで「左が重要」なら、左を先に出して右を後から開くのが正しい順序

### 競合（Re-start）分析とターゲット設計の再確認
- **決定**: Re-start のトップ動画（ビジネス会議シーン）は彼らのモデルでは正解。LinkWorks には参考にしない
- **理由**: Re-start は「事業所」と「企業」の両方に向けたBtoB型プラットフォーム。LinkWorks は就労を考えている個人・家族がターゲットであり、モデルが根本的に異なる
- **教訓**: 競合サイトを参考にする前に「誰に売っているか」を確認する。ターゲットが違えば正解のビジュアルも逆になる
