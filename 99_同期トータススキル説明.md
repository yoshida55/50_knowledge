# 🛡️ RESTART課題 2リモート運用 シンプル版

> 3分で思い出せるように、最小限だけ書いてる。

---

## 🎯 やってること（1行）

**家でこっそり作業 → 翌朝会社で「会社時間」に書き換えてRESTARTに送る**

これだけ。

---

## 📍 場所4つだけ覚える

```
[ 🌐 RESTART ]   公式リポ。先生に提出する場所
     ↑↓
[ 🏢 会社PC ]   ← ここが全ての中継地点（一番偉い）
     ↕
[ 💾 origin ]   自分のGitHub（バックアップ）
     ↑↓
[ 🏠 家PC ]    
```

**ポイント：家は restart に直接触らない。必ず origin 経由。**

---

## 🕐 初回だけ（一度やったらもうやらない）

| いつ | コマンド | 何が起きる |
|------|---------|-----------|
| 新PC到着時 | `/pc-init` | git config / gh CLI / 認証チェック（PC毎1回）|
| 案件開始時（会社）| `/setup-project` （c）| restart から clone → origin を後で追加 |
| 案件開始時（家）| `/setup-project` （h）| origin から clone → restart を後で追加 |

⚠ 家の /setup-project は **会社で1回 /work-end してから** やる（origin が空だと clone できない）

---

## 🕐 1日の流れ（覚えるのこれだけ）

| 時間 | 場所 | コマンド | 何が起きる |
|------|------|---------|-----------|
| 朝 | 🏢会社 | `/work-start` | restart → origin の順で取り込む |
| 夕方 | 🏢会社 | `/work-end` | squashして restart と origin にpush |（またこのタイミングで、当日のコミットのハッシュが登録）
| 夜 | 🏠家 | `/sync-home-start` | origin から強制同期 |
| 寝る前 | 🏠家 | `/sync-home-end` | origin に push（restart には触らない）|

⚠ /work-start の順序：**restart が先、origin が後**（逆だと競合しやすい）

---

## 💡 たった1つの魔法：「squash」

```
家のコミット（時刻バレる）
22:00 「ヘッダー」
23:30 「メイン」
24:00 「フォーム」
        ↓ 翌日の帰り際 /work-end
🪄 squash で1つに統合
        ↓
14:45 「サイト改修」  ← 会社時間に書き換わる ✨
（/work-end は 15:00 前に実行するのがルール）
```

**これが全ての肝**。あとは仕組みが勝手にやる。

---

## 🚫 これだけ守る（4つ）

```
🔴 origin を Public にしない（必ず Private）
🔴 restart/main に直接 push しない（hookで自動防御済）
🔴 commit_all.ps1 に案件フォルダ追加しない
🔴 コミットメッセージに「AI」「Claude」入れない
```

---

## 🆘 困ったら一言

| 状況 | Claude に言う言葉 |
|------|-----------------|
| 競合（conflict） | 「競合解決して」 |
| push が reject | 「push が reject された」 |
| 状態がよくわからない | 「状態確認して」 |
| 全部やり直したい | 「家PCで /setup-project (h) から再セットアップ」 |

---

## 📦 スキル6個の役割

```
⚙️ /pc-init         … PC1台につき1回（新PC時）
📦 /setup-project   … プロジェクト毎に1回（案件開始時）
☀️ /work-start      … 毎朝（会社）
🌙 /work-end        … 毎夕（会社）
🏠 /sync-home-start … 毎晩（家）
🌃 /sync-home-end   … 寝る前（家）
```
