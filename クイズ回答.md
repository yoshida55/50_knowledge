選択されたテキストにある `background-attachment: fixed;` は、背景画像をスクロールさせずに固定するプロパティであり、背景を「透けさせる（透明度を変える）」ためのものではありません。

背景を透けさせるには、一般的に opacity プロパティや、色の指定に rgba() を使用します。それぞれの具体的な使い方を説明します。
1. 要素全体を透けさせる (opacity)
要素（文字も背景もすべて）の透明度を一括で変更します。値は 0（完全に透明）から 1（不透明）の間で指定します。

.box {
  background-color: blue;
  opacity: 0.5; /* 50%の透明度 */
}
2. 背景色だけを透けさせる (rgba)
中の文字などはくっきり見せたまま、背景の色だけを透けさせたい場合に最もよく使われる方法です。

.box {
  /* rgba(赤, 緑, 青, 透明度) */
  background-color: rgba(0, 0, 255, 0.5); /* 青色を50%の透明度に */
}
3. 背景画像だけを透けさせる
背景画像（background-image）自体に透明度を設定するプロパティは存在しないため、擬似要素（::before など）を背後に敷いて、その opacity を調整するのが一般的です。

.box {
  position: relative;
  z-index: 0;
}

.box::before {
  content: "";
  position: absolute;
  top: 0; right: 0; bottom: 0; left: 0;
  z-index: -1;
  background-image: url('image.jpg');
  background-size: cover;
  opacity: 0.3; /* 画像の濃さを30%にする */
}


補足：background-attachment: fixed; とは？
これは、画面をスクロールしても背景画像が一緒に動かず、その場に留まるような視覚効果（パララックス効果など）を作るために使われるプロパティです。透明度とは別の機能です。

★背景を固定する2つの方法の使い分け：
- 普通の背景固定 → `background-attachment: fixed`（こっちが基本・簡単）
- 動く背景が必要な時だけ → `position: fixed` + `z-index`（汎用的）

**Q: 相続税の基礎控除額は？**

3,000万円 + 600万円 × 法定相続人の数

このコードは、**日本の相続税における基礎控除額の計算式**を示しています。

**内容:**
- 相続税には一定額まで課税されない「基礎控除」があります
- その計算式は：**3,000万円 + (600万円 × 法定相続人の数)**

**具体例:**
- 法定相続人が3人の場合：3,000万円 + 600万円 × 3 = **4,800万円**
- 相続財産がこの基礎控除額以下なら相続税はかかりません

**Q: JavaScriptバブルソートの交換条件は？**

## バブルソートの交換条件

**交換条件：`arr[j] > arr[j + 1]`**

左側の要���が右側の要素より大きい場合に、2つの要素を入れ替えます。これにより、大きい値が徐々に右側（配列の末尾）へ移動していきます。

## iとjの流れ

**外側ループ（i）：**
- 配列全体を何回走査するかを制御
- `i = 0, 1, 2, ...` と増加
- i回目のループで、後ろからi番目までがソート済みになる

**内側ループ（j）：**
- 隣接する要素を比較する位置を制御
- `j = 0` から `arr.length - 1 - i` まで
- 各ステップで `arr[j]` と `arr[j+1]` を比較し、必要に応じて交換
- iが増えるごとに比較範囲が狭まる（既にソート済みの部分は比較不要）

**動作例：** `[5, 3, 8, 2]` の場合
- i=0: 全体を比較 → 最大値8が末尾へ
- i=1: 末尾を除いて比較 → 2番目に大きい値が後ろから2番目へ
- i=2: さらに範囲を狭めて比較

**Q: `position: fixed` でレイヤー化する際に、親要素の `z-index` が関係するのはなぜですか？**

`position: fixed` の要素も、親要素に `z-index` が設定されていると、その親のスタッキングコンテキスト内に閉じ込められるため。

**Q: `100vw`と`100%`でスクロールバーの扱いに違いがあるのはなぜ？**

```css
html { font-size: calc(10 / 1280 * 100vw); }
#work {
  width: 92rem;    /* スクロールバー分大きい */
  margin: 0 auto;  /* margin-right: -15px に */
}

/* OK: %指定 */
#work {
  width: 100%;
}
```

幅指定には100%を使えば安全

**Q: CSS maskアニメーション：初期状態をCSSで仕込み、JSでクラス付与してアニメ発動（diagonal-reveal パターン）

【結論】
CSSで「見えない初期状態」と「アニメーション定義」を仕込んでおき、JSが `.is-visible` を付けた瞬間にアニメーションが発動する。

- `@keyframes diagonalReveal` → アニメーション定義（0% → 250%）
- `.diagonal-reveal.is-visible` → JSがクラス付与 → animation 発動

流れ：
1. 初期: `mask-size: 0%` で要素は見えない
2. JSが `.is-visible` を付与（例: スクロールで画面内に入った時）
3. `animation: diagonalReveal` が発動
4. `mask-size` が `0%` → `250%` に変化し、斜め方向に画像が現れる

【具体例】
```css
/* ① アニメーション定義（名前は自由） */
@keyframes diagonalReveal {
  from { mask-size: 0% 0%; }
  to   { mask-size: 250% 250%; }
}

/* ② 初期状態（見えない） */
.diagonal-reveal {
  mask-image: linear-gradient(135deg, black 50%, rgba(0,0,0,0.3) 65%, transparent 80%);
  mask-size: 0% 0%;        /* ← これで非表示 */
  mask-position: 0% 0%;
  mask-repeat: no-repeat;
}

/* ③ JSがis-visibleを付けたら発動 */
.diagonal-reveal.is-visible {
  animation: diagonalReveal 3.3s cubic-bezier(0.4, 0, 0.2, 1) forwards;
}
```

```
初期状態         → アニメ中        → 完了
┌────────┐      ┌────────┐      ┌────────┐
│        │      │▓▓░░    │      │▓▓▓▓▓▓▓▓│
│ 非表示  │  →   │▓▓▓░░   │  →   │▓▓▓▓▓▓▓▓│
│        │      │ ▓▓░░   │      │▓▓▓▓▓▓▓▓│
└────────┘      └────────┘      └────────┘
mask:0%         mask:途中        mask:250%
```

【補足】
- `@keyframes` の名前（`diagonalReveal`）は任意。自分で自由に付けられる
- `forwards` を付けないとアニメ終了後に初期状態（非表示）に戻ってしまう
- `135deg` → 左上から右下へ斜めに表示。角度を変えれば方向も変わる
- `250%` → 100%だと端まで届かない場合があるため大きめに設定
- `-webkit-mask-*` はSafari対応用のプレフィックス。mask-* と両方書くのが安全
- このパターンは mask 以外にも使える（opacity, transform などでも同じ「初期状態 + JS付与 + animation」の構造）

**Q: Grid/Flexレイアウトで画像が変形するのを防ぐために、画像のwidthプロパティに指定すべき値は？**

`width: 100%`

```css
  height: auto;      /* 幅に合わせて高さも自動調整（比率維持） */
  aspect-ratio: 3 / 2;   /* 【重要】横3:縦2 の比率で枠を確保する */
  object-fit: cover; /* 枠からはみ出る分はトリミングして埋める */
  display: block;    /* 画像下の謎の余白を消すおまじない */
}
```

**Q: WordPressにおいて、タイトルを直接出力せず変数に代入可能な関数はどれか？**

**Q: CSSにおいて、コロン1つ（:）の擬似クラスとコロン2つ（::）の擬似要素の役割の違いは何ですか？**

コロン1つは「状態・位置の条件」を指定する擬似クラス、コロン2つは「HTMLにない要素を生成」する擬似要素です。

擬似クラスは対象の状態や位置を絞り込み、擬似要素はコンテンツ自体を新たに作り出す役割があります。

例：
```css
/* 擬似クラス（条件） */
:last-child { ... }

/* 擬似要素（生成） */
::before { ... }
```

**Q: CSS flexの親要素の中で、子要素を中央に寄せるためにmargin: autoを使用するとどうなる？**

flexの子要素にmargin: autoを設定すると、余った空間を左右や上下に均等に割り振り、その要素を中央へ寄せます。

justify-content: centerとは異なり、特定の子要素1つだけを中央に配置したい場合に最適です。flex内ではmargin: autoが「余白の分配器」として機能します。

例：
```css
.news_info {
  margin: auto;
}
```

**Q: 背景を固定したいとき、セクション内だけならsticky、画面全体に固定するなら何を使う？**

画面全体に固定するなら `fixed` を使います。

`sticky` は親要素の範囲内だけ固定されますが、`fixed` は画面そのものに固定されるため、セクションをまたいでも背景を残せます。

例：
```css
.particle_bg {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 100vh;
  background: url("../img/bg.jpg") center / cover no-repeat;
}
```

ただ、z-indexでレイヤーをわけるだけ

【具体例2：トップ背景を固定して下の全セクションを流す（fixed）】
```html
<div class="particle_bg"></div>

<main class="parallax_container">
  <section class="about_area">...</section>
  <section class="news_area">...</section>
  <section class="contact_area">...</section>
</main>
```

```css
/*
  構造:
  <div .particle_bg>            ← 画面全体に固定する背景
  <main .parallax_container>    ← 背景1画面分下から始まる全体コンテンツ
    <section .about_area>       ← 背景の上を流れる各セクション
    <section .news_area>
    <section .contact_area>
*/
.particle_bg {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 100vh;
  z-index: 0;
  background: url("../img/bg.jpg") center / cover no-repeat;
}

.parallax_container {
  position: relative;
  z-index: 1;
  margin-top: 100vh;
  background: #fff;
}
```

# html

**Q: clip-pathを適用した要素の子要素で、z-indexが無効化される原因は何ですか？**

clip-pathが新しいスタッキングコンテキストを生成し、子要素のz-indexが親の枠内に制限されるためです。

`ようするに「スタッキングコンテキスト」ができるっていうこと。`

親要素自体がスタッキングコンテキストを生成することで子要素の階層が固定されるため、解決には親要素自体のz-indexや重なり順を上げる必要があります。

例：
```css
/* 親要素の重なり順を上げる */
.parent {
  clip-path: circle(50%);
  position: relative;
  z-index: 2; /* 子要素より先に親の順序を指定 */
}
```
