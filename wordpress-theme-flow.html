archive.php — 一覧ページ
📋

archive.php

/news/ にアクセスしたときに表示される一覧ページのテンプレート
一覧ページ 手動で作成

作業フロー

1
functions.php にURLを登録
has_archive = 'news' を書く（一回だけ）
⚙️ functions.php を直接編集
2
archive.php をテーマフォルダに作成
VS Code でテーマフォルダに archive.php を新規作成
📁 テーマフォルダ内に新規ファイル作成
3
archive.php に表示処理を書く
ヘッダー読込・投稿を繰り返し取得・フッター読込
📄 archive.php を編集
4
パーマリンクを更新
管理画面 → 設定 → パーマリンク → 保存
🖥 管理画面で操作
✅ /news/ にアクセス → archive.php が自動で呼ばれる

archive.php の中身（最小構成）

<?php get_header(); ?>

<main>
  <h1>お知らせ一覧</h1>

  <?php if ( have_posts() ) : ?>
    <?php while ( have_posts() ) : the_post(); ?>

      <article>
        <a href="<?php the_permalink(); ?>">
          <?php the_title(); ?>
        </a>
        <?php the_date(); ?>
      </article>

    <?php endwhile; ?>
  <?php endif; ?>

</main>

<?php get_footer(); ?>
  • get_header()header.php を読み込む
  • have_posts()投稿があるか確認
  • the_post()次の投稿を取り出す
  • the_title()記事タイトルを表示
  • the_permalink()記事のURLを取得
  • get_footer()footer.php を読み込む
single.php — 記事詳細ページ
📄

single.php

記事タイトルをクリックしたときに表示される1件詳細のテンプレート
詳細ページ 手動で作成

作業フロー

1
functions.php への追加は不要
WordPressが自動でURLを生成してくれる
✅ 何もしなくてOK
2
single.php をテーマフォルダに作成
VS Code でテーマフォルダに single.php を新規作成
📁 テーマフォルダ内に新規ファイル作成
3
single.php に表示処理を書く
タイトル・本文・日付などを表示
📄 single.php を編集
✅ 記事URLにアクセス → single.php が自動で呼ばれる
💡 URLはWordPressが自動生成する
/news/記事スラッグ/ のような形で自動でできる。

single.php の中身（最小構成）

<?php get_header(); ?>

<main>
  <?php if ( have_posts() ) : ?>
    <?php while ( have_posts() ) : the_post(); ?>

      <article>
        <h1><?php the_title(); ?></h1>
        <time><?php the_date(); ?></time>
        <div><?php the_content(); ?></div>
      </article>

    <?php endwhile; ?>
  <?php endif; ?>
</main>

<?php get_footer(); ?>
  • the_title()この記事のタイトルを表示
  • the_date()投稿日を表示
  • the_content()本文（エディタに書いた内容）を表示
⚡ archive.php との違い: the_content() が追加されている（一覧では本文は出さない）

functions.php — 全体設定ファイル
⚙️

functions.php

テーマ起動時に必ず読まれる。URL登録・機能追加をここで行う
全体設定 最初からある

functions.php の役割

新しいURLを生やす has_archive
カスタム投稿タイプを作る register_post_type
ナビメニューを使えるようにする register_nav_menus
CSSやJSを読み込む wp_enqueue_style
WordPressの設定を変更する add_filter

今回のコード（has_archive の場合）

function set_post_archive($args, $post_type) {
  if ( 'post' == $post_type ) {
    $args['has_archive'] = 'news';
    // ↑ /news/ というURLを生やす
    $args['label']       = 'お知らせ';
    // ↑ 管理画面での名前
  }
  return $args;
}
add_filter(
  'register_post_type_args',
  'set_post_archive', 10, 2
);
⚠️ 変更後は必ず
管理画面 → 設定 → パーマリンク → 保存

⚠️

PHPファイルは管理画面から作れない

archive.php / single.php は VS Code などのエディタでテーマフォルダに直接作成する。

✅

PHPファイルは「型紙」

一回作れば全記事に使い回せる。投稿を追加するたびにPHPを書き直す必要はない。

⚠️

archive.php だけではURLは存在しない

functions.php で has_archive を設定して初めて /news/ というURLができる。両方セットで動く。

✅

single.php / page.php は登録不要

WordPressの標準機能として最初から動く。functions.php への追記は不要。

🔄

設定変更後はパーマリンク更新

functions.php を変えたら必ず管理画面 → 設定 → パーマリンク → 保存を実行する。

📋

共通の書き方パターンがある

get_header() → ループ → get_footer() という構成はどのテンプレートも同じ。
