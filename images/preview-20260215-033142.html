display: contents と nth-child の問題

問題の原因

display: contents を使うと、その要素が「透明」になり、子要素が直接親要素の子として扱われます。そのため、nth-child() のカウントが想定外の動作をします。

⚠️ コンテナが「透明化」されると、nth-child() は親要素から直接数えるため、意図しない複数の要素が選択される

解決策：ネストで明示的に指定

共通クラス名を使う場合、ネスト（親からの階層指定）で対象を絞り込むことで、意図した要素のみを選択できます。

/* ❌ 曖昧な指定（複数該当する可能性） */
.共通クラス:nth-child(2) {
  display: none;
}

/* ✅ ネストで明示的に指定 */
.weekly_area .weekly_container .weekly_info_container .radius_link {
  display: none;
}
まとめ
  • display: contents は要素を「透明化」し、子要素が直接親の下に配置される
  • nth-child() は親からのカウントなので、意図しない要素が複数選択される
  • 解決策： 親要素から階層を明示的に指定（ネスト）して対象を絞り込む
