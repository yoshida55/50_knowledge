# フェードイン「スクロールで要素を下から上に表示」- IntersectionObserver版

jQuery不要。要素が画面に入ったときだけ発火するので軽い。

★ `js_soft_reveal` クラスをつけるだけで発動する

## ★重要ポイント

【★ポイント1: CSSで初期状態と表示状態を定義】
```css
.js_soft_reveal {
  opacity: 0;
  transform: translateY(2rem);
  transition: opacity 1s, transform 1s;
}

.js_soft_reveal.is_visible {
  opacity: 1;
  transform: translateY(0);
}
```
`is_visible` クラスが付いたときだけ表示状態になる。JSはクラスをつけるだけでよい。

【★ポイント2: JSはクラスを付けて監視を止めるだけ】
```js
const softRevealObserver = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        entry.target.classList.add("is_visible"); // クラスをつける
        softRevealObserver.unobserve(entry.target); // 1回だけ発動
      }
    });
  },
  { threshold: 0.4 }, // 40%見えたら発動
);

document.querySelectorAll(".js_soft_reveal").forEach((el) => softRevealObserver.observe(el));
```

## 旧jQuery版との比較

以前は jQuery の `$(window).scroll()` を使っていたが、これはスクロールするたびに**全要素をループしてチェック**するので、要素が多いほど重くなる。
また `scroll > target - windowHeight + 200` という計算式を自分で書く必要があり、ミスも起きやすかった。

IntersectionObserver はブラウザが「要素が画面に入ったか」を自動で判定してくれる。
スクロールのたびに処理が走らないので軽く、計算式も不要。jQuery自体を読み込まなくてよくなる。

| | jQuery版 | IntersectionObserver版 |
|--|--|--|
| jQuery | 必要 | 不要 |
| 発火タイミング | スクロールのたびに全要素ループ | 画面に入ったときだけ |
| 計算式 | `scroll > target - windowHeight + 200` | なし（ブラウザが自動判定） |
| コード量 | 多い | 少ない |
| パフォーマンス | 重くなりやすい | 軽い |

> 新しく書くときは迷わず IntersectionObserver を使う。
