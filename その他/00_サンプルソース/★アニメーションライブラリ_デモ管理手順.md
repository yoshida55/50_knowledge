# アニメーションライブラリ デモ管理手順

★ Swiper, GSAP, anime.js等のアニメーションライブラリのデモを統一管理する手順

## 📋 今回やったこと（Swiper.js）

### 1️⃣ デモファイルの整理
- **説明パネル削除**: デモHTML内の `.info-panel`、`.simple-notice` 等を全削除
- **理由**: クライアントに見せる際に説明不要。クリーンなデモのみ
- **説明の配置**: `index.html` のカードに集約（開発者向け情報）

### 2️⃣ デモの並び替え
- **旧**: サムネイル → 自動再生 → 多機能 → シンプル
- **新**: シンプル → サムネイル → 自動再生 → 多機能
- **理由**: 学習効率（基礎→応用の順）

### 3️⃣ ファイル名規則の確立
- **❌NG**: `swiper_basic_to_advanced.md` （抽象的）
- **✅OK**: `swiper_フェード切替.md` （具体的）
- **ルール**: `{ライブラリ名（英語）}_{アニメーションタイプ（日本語）}.md`

## 🎯 今後使うコマンド

### アニメーションライブラリのデモ生成
```bash
/animation-library-snippet {ライブラリ} {タイプ}
```

**例**:
```bash
/animation-library-snippet swiper フェード切替
/animation-library-snippet gsap スクロール連動
/animation-library-snippet anime パララックス
```

## ✅ 必須ルール

### デモ構成（常に4つ）
1. **シンプル版**（必須・最初）- ライブラリの基本機能のみ
2. **パターン1** - よく使う実装例
3. **パターン2** - よく使う実装例
4. **パターン3** - 多機能応用例

### ファイル名
- ライブラリ: 英語（swiper, gsap, anime）
- タイプ: **日本語**（フェード切替, スクロール連動, パララックス）
- MDファイル: `★` で開始

### パス指定
- MD内リンク: **相対パス** `../../images/`
- 理由: 会社・自宅で絶対パスが異なるため

### クライアント対応
- デモHTML: 説明パネル **禁止**
- 説明: `index.html` のカードのみ
- 理由: クライアントに見せる際にクリーンな印象

## 💡 運用フロー

### 新規ライブラリ追加時
```
1. /animation-library-snippet {library} {タイプ}
2. 生成確認
   - MDファイル
   - 4つのデモHTML
   - index.html更新
3. 動作確認
   - 各デモHTMLをブラウザで開く
   - index.htmlから遷移確認
```

### 既存ライブラリ追加デモ
```
1. 既存のデモHTMLを参考に手動作成
2. index.htmlに新規カード追加
3. MDファイルにリンク追加
```
