# ホバー時の切り替え表示 - テキストを上に非表示にし下からテキストを表示する（疑似要素スライド）

親要素にホバーすると、テキストが上にスライドして消え、同じテキストが下からスライドして表示されるアニメーション。

★枠（overflow: hidden）は動かさず、疑似要素（::before / ::after）だけを動かす

！！！！！以下の書き方を覚える　ホバー要素と、実際動作するホバーをかえることができる
```css
.parent:hover .text::before { transform: translateY(-100%); }
.parent:hover .text::after  { transform: translateY(0%); }
```

## HTML
```html
<!-- data-text に表示テキストを設定 -->
<!-- 実テキストは空でもOK（color: transparent で非表示のため） -->
<div class="summary_container">

  <a href="#" class="view_more_btn">
    <!-- ★ data-text = ::before / ::after が表示するテキスト -->
    <span class="view_more_text" data-text="View More"></span>
    <span class="arrow_icon">→</span>
  </a>

</div>
```

---

## ★重要ポイント

クリップ = 「はみ出した部分を切り取って見えなくする」こと

【★ポイント1: overflow: hidden + transform の罠】
```css
/* ❌ ダメな例：要素に対して、(transform: translateY(-100%))枠ごと動く → ::after がクリップされて見えない */

.view_more_text {
  overflow: hidden;
  transition: transform 0.3s ease;
}
.view_more_text:hover {
  transform: translateY(-100%); /* 枠ごと上に移動してしまう */
}


/* ✅ 正しい：枠は固定、中の疑似要素（テキストの疑似要素を作成して）それを動かす */
.view_more_text {
  overflow: hidden; /* 枠 = 動かない */
}
.parent:hover .view_more_text::before {
  transform: translateY(-100%); /* 中身だけ動く（対象は疑似要素の文字） */
}
```
`overflow: hidden` は枠と一緒に動く → 枠自体を `transform` してはいけない




【★ポイント2: ::before + ::after で枠固定・中身だけ動かす構造】
```css
/* 実テキストを透明にして、疑似要素で表示する */
.view_more_text {
  color: transparent;  /* 実テキスト非表示 */
  overflow: hidden;    /* はみ出し禁止の枠 */
  position: relative;  /* ::before/::after の基準点 */
  height: 2rem;        /* 固定値必須（autoだとクリップ不可） */
  line-height: 2rem;   /* テキスト縦中央 */
}

/* 通常表示用（見える位置） */
.view_more_text::before {
  content: attr(data-text);
  position: absolute;
  top: 0; left: 0;
  color: black;
  transition: transform 0.3s ease;
}

/* ホバー表示用（下で待機） */
.view_more_text::after {
  content: attr(data-text);
  position: absolute;
  top: 0; left: 0;
  color: black;
  transform: translateY(100%); /* 下に隠す */
  transition: transform 0.3s ease;
}
```
枠（.view_more_text）は固定、::before と ::after だけが枠の中をスライドする



【★ポイント3: ホバートリガーは親要素にかける】
```css
/* カード全体にホバーしたら子の疑似要素を動かす */
.summary_container:hover .view_more_text::before {
  transform: translateY(-100%); /* 上へ退場 */
}
.summary_container:hover .view_more_text::after {
  transform: translateY(0%);    /* 下から登場 */
}
```
`親:hover 子::before` = 「親にホバーしたら → 子の中の疑似要素を動かす」




【★ポイント4: height は固定値必須】
```css
/* ❌ auto → クリップの境界が定まらない */
.view_more_text { height: auto; overflow: hidden; }

/* ✅ 固定値 → クリップが正しく機能する */
.view_more_text { height: 2rem; overflow: hidden; }
```
`overflow: hidden` のクリップには固定の高さが必要

## CSS
```css
/* ========================================
   ホバー時テキストスライド切り替え
   - 枠（overflow: hidden）は固定
   - ::before（通常表示） → 上へ退場
   - ::after（ホバー表示） → 下から登場
   ======================================== */

/* --- 親コンテナ --- */
.view_more_btn {
  position: absolute;
  width: 15rem;
  bottom: 1rem;
  left: 3rem;
  display: flex;
  align-items: center;
  gap: 0.5rem;
  z-index: 15;
}

/* --- 枠（固定・動かない） --- */
.view_more_text {
  display: block;
  width: 70%;
  height: 2rem;          /* ★ 固定値必須（autoだとクリップ不可） */
  line-height: 2rem;     /* テキスト縦中央 */
  font-size: 2rem;
  font-weight: bold;
  color: transparent;    /* ★ 実テキスト非表示（疑似要素が代わりに表示） */
  overflow: hidden;      /* ★ はみ出し禁止の枠 */
  position: relative;    /* ★ ::before/::after の基準点 */
}

/* --- 通常表示テキスト（::before） --- */
.view_more_text::before {
  content: attr(data-text);  /* ★ data-text属性からテキスト取得 */
  position: absolute;
  top: 0;
  left: 0;
  color: black;
  transition: transform 0.3s ease;
}

/* --- ホバー表示テキスト（::after）--- */
.view_more_text::after {
  content: attr(data-text);  /* ★ 同じテキスト */
  position: absolute;
  top: 0;
  left: 0;
  color: black;
  transform: translateY(100%);  /* ★ 下で待機 */
  transition: transform 0.3s ease;
}

/* --- ホバー時：親にホバーしたら疑似要素を動かす --- */
.summary_container:hover .view_more_text::before {
  transform: translateY(-100%);  /* ★ 上へ退場 */
}

.summary_container:hover .view_more_text::after {
  transform: translateY(0%);     /* ★ 下から登場 */
}
```



---

## 動作フロー
1. 通常時：`::before` が `translateY(0)` で表示、`::after` が `translateY(100%)` で下に隠れている
2. 親要素にホバー：`::before` が `translateY(-100%)` で上に退場、`::after` が `translateY(0)` で下から登場
3. ホバー解除：`transition` で元の位置に戻る

## トラブルシューティング
### エラー: ホバーしてもテキストが消えない
**原因**: `overflow: hidden` が無い、または `height: auto` になっている
**対処**: `overflow: hidden` + `height` を固定値に設定

### エラー: ::after がホバー前から見えてしまう
**原因**: `overflow: hidden` が無い
**対処**: `.view_more_text` に `overflow: hidden` を追加

### エラー: ホバーで枠ごと動いて::afterも消える
**原因**: `.view_more_text` 自体に `transform` をかけている
**対処**: `.view_more_text` の `transform` を削除し、疑似要素だけを動かす
